# Data Lineage

## 1. Data Sources

Current canonical dataset is built from two raw sources:

1. `data/fixtures/sample_movies.csv`
- curated structured movie rows
- includes title/year/genres/rating/vote_count/overview and related metadata

2. `data/fixtures/tmdb_sample_movies.json`
- TMDB-style JSON payload (`genres` + `results`)
- can emulate API ingestion behavior without network dependency

Optional source:
- TMDB live API via `ENABLE_TMDB_API=true` + `TMDB_API_KEY`

## 2. Lineage Stages

### Stage A: Ingest
- Files copied to `data/raw/` by `scripts/seed_sample_data.py`.
- Ingestion pipeline loads CSV and TMDB-style JSON into normalized in-memory frames.

### Stage B: Clean
- Title normalization: lowercase + alphanumeric normalization for joinability.
- Release year harmonization: prefer explicit year; fallback parse from release_date.
- Genre harmonization: pipe-separated canonical genre string.
- Missing handling: fill text defaults, numeric coercion, safe null handling.
- Deduplication within each source: key = `(normalized_title, release_year)`.

### Stage C: Merge
- Sources merged on `(normalized_title, release_year)`.
- Reason: IDs differ by source; title alone is ambiguous; title+year is practical coursework key.
- Source provenance retained via `source_flags` (e.g., `csv|tmdb`).

### Stage D: Canonical Dataset
- Canonical movie rows saved to:
  - `data/processed/05_merged_movies.csv`
- Loaded into SQLite `movies` table.

### Stage E: Document Generation
- Document per movie built from:
  - title
  - genres
  - overview
  - tagline
  - review snippet
- Saved to:
  - `data/processed/06_movie_documents.csv`
- Loaded into SQLite `movie_documents` table.

### Stage F: Embedding + Vector Index
- Retriever reads `movie_documents` from SQLite.
- Embeddings generated by:
  - sentence-transformers (preferred)
  - deterministic hash fallback (if dependency unavailable)
- Vectors persisted in local vector store:
  - Chroma (preferred)
  - JSON fallback store

## 3. Lineage Guarantees

- All pipeline stages are local and reproducible.
- Input and output artifacts are persisted under `data/raw/` and `data/processed/`.
- Agent answers are grounded on SQLite + vector index evidence, with explicit evidence titles.
